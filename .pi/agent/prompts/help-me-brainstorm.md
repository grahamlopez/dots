Let's brainstorm together and make a plan.

$@

To make this brainstorming session really great, do:
- Present various alternatives along the way and present them to the user for consideration. Give pros and cons for each, and tell the user which you recommend.
- Make sure you understand the entire problem to your own satisfaction without any gaps. Ask the user to clarify anything that is vague.
- Consider the ideas in the context of implementation complexity and project maintenance. Are there serious tradeoffs? Discuss with the user.

The user may have more questions and ideas after answering your first questions, so allow for some iteration and back and forth. After questions have been answered and back and forth is finished, verify with the user that we are ready to make a plan.

Once we are satisfied with the general direction from the brainstorming, we will make a plan. This project values above all else implementation simplicity and reuse of available framework/library components and common well-understood patterns in an effort to have the highest possible long-term maintainability for agentic developers. When making your plan, carefully analyze and comment about:
- Custom or overly complex implementation
- Code brevity and understandability
- Code duplication of any kind
- Test coverage and potency/effectiveness
